## 2. Диагностика: почему контроллер ломает img

### 2.1 Три режима работы (из TensorBoard trajectories)

**E6 (линейный контроллер) — спираль смерти img:**
```
w_img:            1.00 → 0.30   (контроллер режет вес img в 3×)
loss_img_contrast: 3.04 → 2.47  (img лосс не снижается)
img→centroid R@1:  0.90 → 0.12  (катастрофа)
```
Механизм: img-лосс выше текстовых → контроллер снижает w_img → img перестаёт учиться
→ лосс остаётся высоким → ещё снижает w_img. Death spiral.

**E8c (нелинейный контроллер) — мягче, но D_intra убивает:**
```
w_img:            1.00 → 0.90   (нелинейный more conservative)
D_intra:          0.63 → 0.04   (alignment + radial сжимают пространство)
img→centroid R@1:  0.61 → 0.37  (img не успевает за сжатием)
λ_rad:            0.10 → 0.04   (контроллер пытается помочь, но поздно)
```

**E1 (без alignment, без контроллера) — здоровый рост:**
```
D_intra:          3.20 → 1.28   (пространство "свободное")
img→centroid R@1:  0.08 → 0.77  (стабильный рост)
cm_R@1:           0.00 → 0.44   (лучший cross-modal)
```

### 2.2 Корневые причины (три)

**Причина 1: Alignment loss (L_align) + D_intra → 0 ↔ cm_R@1 — антагонисты.**
- L_align тянет e_m → centroid. При D_intra→0.015 все модальности "на центроиде"
- mod→centroid идеален (0.97), но объекты неразличимы по отдельной модальности
- E1 без alignment держит D_intra=1.28 и выигрывает cm_R@1

**Причина 2: Контроллер E6 — no floor protection для слабых модальностей.**
- Видит что img-лосс высокий → снижает w_img → img перестаёт учиться
- Нет механизма w_min: вес может уйти в 0.30, уничтожив модальность
- E8c (нелинейный) ослабляет проблему (w_img до 0.90 только), но не решает

**Причина 3: img — принципиально другая природа (CNN vs Transformer).**
- 4 текстовых модальности = один backbone (SciRus-tiny), схожие representations
- img = ResNet/ConvNeXt, representations в другом подпространстве
- Центроид bias к текстовым: c = (4×text + 1×img)/5, img = 20% влияния
- При alignment давлении img вынужден "догонять" текстовый кластер

### 2.3 Контрольные цифры E3/E4 vs E8c vs E1

| Метрика | E3 (no ctrl) | E8c (ctrl) | E1 (pairwise) |
|---------|-------------|------------|---------------|
| D_intra | 0.015 | 0.041 | 1.283 |
| img→centroid R@1 | 0.944 | 0.374 | 0.773 |
| cm_R@1 | 0.314 | 0.406 | 0.443 |
| collapse_score | 0.0004 | 0.0004 | 0.023 |

E3: всё сколлапсировано к центроиду (D_intra=0.015), идеальный mod→c, но модальности неразличимы.
E8c: промежуточный D_intra, контроллер давит img.
E1: свободное пространство, все модальности растут самостоятельно.

## 3. Это отрицательный результат?

**Частично.** Composite loss (E3-E10) не лучше простого PairwiseInfoNCE для универсальной модели.
Но это **ценный диагностический результат** для Paper B и Paper C:

- **Paper B:** alignment+radial регуляризация создаёт collapse-to-centroid эффект.
  Нужен баланс: mod→centroid хорош, но cm_R@1 страдает.

- **Paper C:** fuzzy controller без floor protection усугубляет дисбаланс модальностей.
  Нелинейный (E8c) лучше линейного (E6), но проблема структурная.

**Для публикации это ablation story:** "мы пробовали сложнее, вот почему проще работает,
вот условия при которых сложнее может работать лучше."

## 4. План sweep13 — гипотезы для проверки

### H51: w_min clamp в контроллере
Добавить `w_min=0.5` в TSFuzzyController: `new_lambda[5:10] = clamp(new_lambda[5:10], min=w_min)`.
Это предотвращает death spiral для img. Ожидание: E6 с w_min не убьёт img→centroid.

### H52: Reduced alignment (λ_align ÷10)
Composite loss с λ_align=0.03 вместо 0.3. D_intra не будет коллапсировать,
сохраняя модальную структуру. Ожидание: E3/E4 с reduced alignment → cm_R@1 ↑.

### H53: E8c + w_min + reduced alignment (комбинация H51+H52)
Нелинейный контроллер с floor protection И ослабленным alignment.
Ожидание: лучший из обоих миров.

### H54: Asymmetric centroid weighting
Центроид c = Σ α_m · e_m с learnable α, инициализированным α_img=0.5, α_text=0.125.
Компенсирует текстовый bias (4 text vs 1 img в усреднении).

### H55: img warmup — поздний старт alignment для img
Первые 1000 шагов: img учится только через L_contrast (pairwise).
После 1000: включаем alignment для img. Даёт img время "войти" в пространство.

### Приоритет: H51 + H52 → H53 → H55 → H54

### Конфиги для sweep13 (предварительные)
```

e3_low_align.yaml    — E3 + λ_align=0.03 (H52)
e4_low_align.yaml    — E4 + λ_align=0.03 (H52)
e8c_low_align.yaml   — E8c + w_min=0.5 + λ_align=0.03 (H53)
e8c_img_warmup.yaml  — E8c + img alignment delayed 1000 steps (H55)
```
