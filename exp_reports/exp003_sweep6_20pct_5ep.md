## 0. Цель

Проверить влияние 4 фиксов визуального пайплайна на cross-modal retrieval img→text.

**Фиксы (все применены одновременно):**
- **P1:** ImageNet нормализация (mean=0.449, std=0.226 вместо 0.5/0.5)
- **P2:** Глубокий AlignNet — MLP(512→GELU→LN→Drop→312→LN) вместо Linear(512→312)
- **P3:** Freeze ResNet conv1+layer1+layer2 + 3 LR группы (text 0.1×, visual 0.5×, heads 1×)
- **P4:** Visual align loss к 3 модальностям [latex, en, lean] вместо только [latex]

**Ожидание:** img→centroid R@1: 0.001 → 0.15+, img→text R@1: 0.000 → 0.05+

## 9. Ключевые выводы

### 9.1 Visual pipeline fix — ПРОРЫВ

Фиксы P1-P4 **полностью решили** проблему img→* ≈ 0:
- img→centroid R@1: **0.001 → 0.84-0.94** (840-940× рост)
- img→latex R@1: **0.000 → 0.30** (E1)
- img→text R@1: **0.000 → 0.08-0.12** (E1, для en/ru/lean)

Compound-эффект: каждый фикс по отдельности дал бы малый прирост. Вместе — качественный скачок.

### 9.2 Ranking экспериментов

**По mean_crossmodal_R@1 (прямой retrieval):**
1. E1 (pairwise): **0.474** — pairwise loss создаёт самые сильные прямые пары
2. E1b (pairwise B): 0.449 — shared encoder немного хуже (−5%)
3. E6 (fuzzy): 0.418 — контроллер даёт лучшую геометрию за счёт cross-modal
4. E7 (lyapunov): 0.417 — ≈ E6, Lyapunov term без эффекта

**По img→centroid R@1 (centroid retrieval):**
1. E3 (centroid+reg): **0.944** — alignment loss максимизирует proximity к центроиду
2. E5 (learnable): 0.942
3. E4 (static): 0.942
4. E2 (centroid only): 0.932

**Парадокс E1 vs E3:** E1 лидирует по cross-modal, но проигрывает по centroid retrieval. E3 — наоборот. Причина: pairwise loss оптимизирует прямые пары (en↔ru, lean↔latex), а centroid loss — proximity к центру.

### 9.3 Family A vs Family B

Family A (5 encoders, 112M params) стабильно лучше Family B (2 encoders, 28M params):
- E1 vs E1b: 0.474 vs 0.449 (−5%)
- E3 vs E3b: 0.353 vs 0.321 (−9%)
- E4 vs E4b: 0.357 vs 0.322 (−10%)

Overhead: 4× параметров → +5-10% по cm_R@1. Для Paper A: separate encoders оправданы.

### 9.4 E2 collapse

E2 (centroid InfoNCE без регуляризации) по-прежнему коллапсирует: mean_cm_R@1 = 0.092. D_intra = 2.15 — эмбеддинги разбегаются вместо схождения к центроиду. Centroid loss без alignment/radial reg — нежизнеспособный конфигурация.

### 9.5 E6 ≈ E7

Lyapunov soft constraint (xi=0.01) в E7 не даёт статистически значимого отличия от E6:
- cm_R@1: 0.418 vs 0.417
- img→centroid: 0.907 vs 0.909
- D_intra: 0.0167 vs 0.0167

Для Paper C: нужно либо увеличить xi, либо показывать E7 как stability guarantee, а не как performance improvement.

### 9.6 Масштаб данных

EXP-002 (10%/3ep) → EXP-003 (20%/5ep): средний прирост +0.082 (+35%). Метрики не насыщены — есть потенциал для дальнейшего роста на 100%/10-15ep.
